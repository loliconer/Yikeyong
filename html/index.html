站点介绍

本站建设的目的在于学习最新的前端技术并将之应用于实际的开发中，从而打造一个超现代化的网站。因此，大部分页面由于采用了新特性只能用最新的浏览器访问。为了促进前端行业的积极发展，本站不对各种低版本浏览器做兼容性处理，如有需要，请自行搜索相关的 Polyfill。推荐使用 Chrome、Firefox、Opera、Safari、Edge 浏览器的最新版本。各种套壳的国产浏览器的内核更新速度实在太慢，也不推荐使用。滚蛋吧，IE！

以下详细介绍本站开发中使用到的各种常见和不常见的黑科技，供大家参考和学习，此列表会不断更新。

一、HTTPS

本站服务器采用 Let's Encrypt 实现 https 加密，并开启了 http2 协议，SSL 协议仅支持 SLV1.1 和 SLV1.2。开启了 http2 以后，如果 ssl_ciphers 使用默认配置，会导致部分浏览器，比如 Chrome47-，无法访问站点，请使用 Mozilla 的推荐配置。

二、纯粹的前后端分离

本站采用完全前后端分离的方式开发，页面全部为静态的HTML，未使用服务器端渲染的方式，通过API异步请求数据。后端使用 Nginx 提供静态资源服务，使用 Restify 提供API服务，使用 Redis 管理 Session。

三、Nginx 的 SSI

由于未使用服务器端渲染HTML页面，因此利用 Nginx 的 SSI 模块的 include 指令将通用的部分包含进来。

四、Fetch

前端所有的异步请求均使用浏览器原生的 fetch 方法。
影响页面：博客、Rosetta Stone、硬件DIY
兼容性：Edge14+、Firefox39+、Chrome42+、Safari10.1+、Opera36+

五、IntersectionObserver

使用该 API 实现图片懒加载和滚动到底部自动加载更多。在 Edge15 中有个 问题 需要注意。
影响页面：博客、精选女头
兼容性：Edge15+、Firefox55+、Chrome51+、Opera46+

六、NodeList 的 forEach

可以直接使用 forEach 方法遍历 document.querySelectorAll 返回的结果。由于 Edge 浏览器也不支持该方法，因此做了兼容性处理，参考 NodeList的forEach方法的兼容性处理。
影响页面：精选女头、Vue 前端组件
兼容性：Firefox50+、Chrome51+、Safari10+、Opera38+

七、Vue SPA

使用 Vue 框架结合 Vuex、Vue-Router 开发的 SPA。目前有：硬件DIY

八、Markdown

本站所有的博客和文章均使用 Markdown 编写，写完之后自动编译成静态的 HTML 页面。语法高亮使用的是 Prism。

九、响应式设计

部分页面实现了响应式设计，方便在移动设备上浏览。

十、ES2015

前端开发中，JavaScript 均采用 ES6+ 的语法和特性，未使用 Babel 编译，且尽量使用已知的各种最佳实践，力求代码的简洁和优雅。
兼容性：IE11+、Edge12+、Chrome49+、Firefox45+、Safari10+、Opera36+
